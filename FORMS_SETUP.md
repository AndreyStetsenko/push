# Настройка форм Contact Form 7

## Быстрый способ установки ID формы

### Вариант 1: Прямо в коде (самый простой)

Откройте файл `wp-content/themes/push/components/home/pushstart.php` и найдите строку:

```php
// $pushstart_form_id = 123; // Замените на реальный ID формы
```

Раскомментируйте и укажите ID вашей формы:

```php
$pushstart_form_id = 123; // Замените на реальный ID формы
```

То же самое для формы в футере в файле `footer.php`:

```php
$formfooter_form_id = 124; // Замените на реальный ID формы
```

### Вариант 2: Через опции WordPress

Добавьте в `functions.php` (временно, в конец файла):

```php
// Установите ЧИСЛОВЫЕ ID ваших форм CF7
update_option('pushstart_cf7_form_id', 123); // Замените на реальный числовой ID
update_option('formfooter_cf7_form_id', 124); // Замените на реальный числовой ID
```

После установки ID можно удалить эти строки.

## Как найти ID формы

1. Перейдите в админке WordPress: **Contact → Contact Forms**
2. Откройте форму для редактирования
3. Посмотрите URL в браузере: `post.php?post=123&action=edit`
4. Число `123` - это и есть ID формы (не хеш из шорткода!)

## Автоматический поиск по названию

Функция автоматически ищет форму по названию, если ID не установлен:
- Для `pushstart` ищет формы с названиями: "pushstart", "push-start", "push start"
- Для `formfooter` ищет формы с названиями: "formfooter", "form-footer", "footer", "form footer"

## Компиляция

После установки ID пересоберите проект:

```bash
npm run build
```

## Проверка

1. Откройте главную страницу
2. Откройте консоль браузера (F12)
3. Выполните: `document.querySelector('.pushstart-form').getAttribute('data-form-id')`
4. Должен вернуться числовой ID формы (не пустая строка)

## Устранение проблем

Если форма все еще перезагружает страницу:
1. Убедитесь, что ID установлен и не пустой
2. Проверьте консоль браузера на ошибки JavaScript
3. Убедитесь, что скрипты скомпилированы (`npm run build`)
4. Очистите кеш браузера (Ctrl+Shift+R)
